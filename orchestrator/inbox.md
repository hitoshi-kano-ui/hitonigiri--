# 家老への申し送り

軍師・各チームからの引き継ぎ・確認事項をここに記載する。
家老は会話開始時にこのファイルを確認し、対応すること。

## 2026-03-14 軍師からの引き継ぎ

### 【決定事項①】コーポレート機能チームの編成
・内容：以下の3チームを家老直下に新設する
  1. コンテンツチーム（SNS・HP・コラム等の発信物全般）
  2. 資料作成チーム（提案書、サービス説明資料）
  3. リサーチチーム（法令・制度、市場・競合、コンテンツ素材）
・背景・理由：代表との壁打ちにより、事業別ではなく機能別編成を採用。顧客層が共通（30〜300名規模企業）であり、代表一人体制では機能別の方が効率的
・期待する成果：コーポレート機能の体制確立
・優先度：高

### 【決定事項②】リサーチチームを最優先で構築
・内容：以下の設計で構築すること
  - 構成：リサーチャー1体（エージェント分割はしない）
  - 業務3本柱：
    1. 法令・制度リサーチ（労働法改正追跡、助成金監視、判例収集）
    2. 市場・競合リサーチ（競合動向、HR Tech市場、顧客層の課題変化）
    3. コンテンツ素材リサーチ（人事トレンド統計・事例、SNSバズ投稿収集）
  - ツール：Web検索、e-gov法令API、SNS API、PDF読み取り
  - アウトプット先：`orchestrator/teams/research/reports/`（legal/ market/ content/）
  - 起動方式：オンデマンド（定期実行は後日検討）
・背景・理由：リサーチは資料作成・コンテンツ制作の前提となるため、最初に立ち上げるべき
・期待する成果：代表や他チームが必要な情報を依頼できる体制
・優先度：高

### 【決定事項③】コンテンツチームの設計
・内容：以下の設計で構築すること（リサーチチーム・旧SNS削除の後に着手）
  - 配置：`orchestrator/teams/content/`
  - 構成：エージェント3体（フェーズ1・最小構成）
    1. エディター（編集長）— 企画・品質管理・トーン統一・代表レビュー窓口
    2. ライター（1体）— SNS投稿・コラム・ブログ・note・コピーライティング全般
    3. ウェブ制作担当（1体）— HP/LP設計・コーディング・画像作成全般
  - 指揮系統：家老 → エディター → ライター / ウェブ制作担当
  - リサーチチームからコンテンツ素材を受け取り、エディターが企画に落とし込む
  - 発信トーンは `context/persona.md` に準拠
  - フェーズ2（運用安定後）：必要に応じてライター・ウェブ制作を専門別に分割
    - 分割判断基準：1体のプロンプトが肥大化し品質が落ちた時
・背景・理由：SNS・コラム・HP等の発信を一元管理し、テーマの一貫性を保つ。最小構成で始め、運用実績に基づいて分割する方針
・期待する成果：全発信チャネルの統合管理体制
・優先度：中（リサーチチーム構築後に着手）

### 【決定事項④】資料作成チームの設計
・内容：以下の設計で構築すること（コンテンツチーム構築後に着手）
  - 配置：`orchestrator/teams/documents/`
  - 構成：ドキュメンター1体
  - 業務：
    1. 提案書テンプレート作成（各事業01〜04のサービス紹介・課題仮説・料金プラン）
    2. デリバリー資料テンプレート作成（HSM診断レポート雛形、制度設計ドラフト雛形、研修資料雛形など）
  - 管轄外：個社向けカスタマイズ（第2層・クライアント別エージェントの管轄）
  - 起動方式：オンデマンド
・背景・理由：テンプレートを整備することで、クライアント対応時の初動を高速化する。個社カスタマイズは第2層が担うため、ここでは雛形に徹する
・期待する成果：各事業の提案書・デリバリー資料のテンプレート一式
・優先度：中（コンテンツチーム構築後に着手）

### 【決定事項⑤】旧SNSチームの削除
・内容：`orchestrator/teams/SNS/` を削除し、コンテンツチームとして再構築する
・背景・理由：SNSチームは箱のみで未稼働。コンテンツチームに統合する方針のため、一度削除して再構築する。代表了承済み
・期待する成果：不要な構成の整理
・優先度：中（リサーチチーム構築後に着手）

### 組織図（決定済み）

```
    ├── コンテンツチーム（新設。旧SNSチーム再構築）
    ├── 資料作成チーム（新設）
    └── リサーチチーム（新設・最優先）
```
